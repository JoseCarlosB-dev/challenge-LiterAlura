# Challenge LiterAlura

Uma aplicação desenvolvida com **Java e Spring Boot** que consome dados de uma API pública de livros e realiza o armazenamento dessas informações em um banco de dados.

## Sobre o Projeto

O **LiterAlura** é um desafio de desenvolvimento que consiste em criar uma aplicação de catálogo de livros capaz de:

* Consumir dados de uma API de livros (Gutendex)
* Processar e manipular os dados recebidos
* Persistir as informações em um banco de dados
* Disponibilizar funcionalidades de consulta para o usuário

O objetivo principal do projeto é aplicar na prática conceitos aprendidos durante os estudos em **Java e Spring Boot**, incluindo consumo de APIs e persistência de dados.

## Funcionalidades

A aplicação possui as seguintes funcionalidades principais:

1. Buscar livro pelo título
2. Listar livros registrados no banco de dados
3. Listar autores registrados
4. Listar autores vivos em um determinado ano
5. Listar livros em um determinado idioma

## Objetivos do Desafio

* Praticar os conceitos aprendidos nos cursos anteriores
* Desenvolver uma aplicação utilizando **Spring Boot**
* Trabalhar com **consumo de API externa**
* Implementar **persistência de dados** em banco de dados
* Consolidar o conhecimento construindo uma aplicação completa

## Tecnologias Utilizadas

* Java
* Spring Boot
* Spring Data JPA
* Hibernate
* Maven
* Banco de Dados Relacional
* API Gutendex

## Como Executar o Projeto

1. Clone o repositório:

```bash
git clone https://github.com/seu-usuario/literalura.git
```

2. Abra o projeto em uma IDE Java, como:

* IntelliJ IDEA
* Eclipse
* Visual Studio Code

3. Execute a aplicação a partir da classe principal do projeto.

## API Utilizada

A aplicação utiliza a **Gutendex API**, uma API pública baseada no projeto Gutenberg que fornece dados sobre livros e autores.

## Considerações

Este projeto foi desenvolvido com foco educacional, com o objetivo de praticar desenvolvimento backend utilizando **Java e Spring Boot**.
